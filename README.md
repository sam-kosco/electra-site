# Electra Site

Static two-page website (Home + Photo Gallery) hosted on GitHub Pages. Photos are
sourced from the DataHub SharePoint site and synced into the repo by a GitHub
Actions workflow that Power Automate can trigger on demand.

## How it works

1. Photos live on SharePoint: `DataHub â†’ Shared Documents â†’ Textbooks/Electra Site/Home Page Pic` and `.../Photo Gallery`.
2. The `Update Photos` workflow (`.github/workflows/update_photos.yml`) runs `sync_photos.py`, which authenticates to Microsoft Graph with the **Foxtrot Report Automation** Entra app, downloads the images into `images/home/` and `images/gallery/`, writes `photos.json`, and commits the result.
3. `index.html` and `gallery.html` read `photos.json` at page load via `script.js`. The home page shows the first image (alphabetically) from the Home Page Pic folder, or nothing if the folder is empty. The gallery carousel shows every image in the Photo Gallery folder.
4. GitHub Pages serves the site; a Power Automate flow triggers the workflow via `repository_dispatch` whenever photos should refresh.

## One-time setup

### 1. GitHub Secrets

Settings â†’ Secrets and variables â†’ Actions. Same three values as the other
Foxtrot automation repos (secrets are per-repo, so they must be added here too):

| Secret | Source |
|---|---|
| `TENANT_ID` | Microsoft Entra tenant ID |
| `CLIENT_ID` | Foxtrot Report Automation app registration |
| `CLIENT_SECRET` | Entra â†’ App registrations â†’ Foxtrot Report Automation â†’ Certificates & secrets |

Note: when the shared `CLIENT_SECRET` is renewed, this repo now needs updating too.

### 2. Enable GitHub Pages

Settings â†’ Pages â†’ Source: **Deploy from a branch** â†’ Branch: `main`, folder `/ (root)`.

The repo must be **public** for free GitHub Pages (anything committed here,
including the photos, is publicly visible on the internet).

### 3. Custom domain (GoDaddy)

In GitHub: Settings â†’ Pages â†’ Custom domain â†’ enter `www.yourdomain.com` and save
(GitHub commits a `CNAME` file to the repo automatically).

In GoDaddy: My Products â†’ DNS for the domain â†’ add these records:

| Type | Name | Value |
|---|---|---|
| CNAME | `www` | `<your-github-username>.github.io` |
| A | `@` | `185.199.108.153` |
| A | `@` | `185.199.109.153` |
| A | `@` | `185.199.110.153` |
| A | `@` | `185.199.111.153` |

Delete any existing GoDaddy "Parked" A record on `@` and any conflicting CNAME on
`www`. DNS can take from a few minutes up to ~48 hours to propagate. Once GitHub
shows the DNS check passed, tick **Enforce HTTPS** on the Pages settings page
(the certificate may take up to an hour to issue).

### 4. Power Automate trigger

Create a fine-grained GitHub Personal Access Token (GitHub â†’ Settings â†’ Developer
settings â†’ Fine-grained tokens) scoped to **only this repo** with
**Contents: Read and write** permission.

In the flow, add an **HTTP** action:

- Method: `POST`
- URI: `https://api.github.com/repos/<your-github-username>/<repo-name>/dispatches`
- Headers:
  - `Accept`: `application/vnd.github+json`
  - `Authorization`: `Bearer <your-PAT>`
  - `User-Agent`: `FoxtrotPA`
- Body:

```json
{ "event_type": "update-photos" }
```

A successful call returns **204 No Content** and the `Update Photos` workflow
appears in the Actions tab within a few seconds. Photos refresh on the live site
about a minute later (GitHub Pages redeploys on the commit).

The workflow can also be run manually: Actions â†’ Update Photos â†’ Run workflow.
